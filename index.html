Cost and Billing

Claude Code is not a flat subscription. Here's how pricing works.

  • Pay-per-use. Claude Code charges by token (roughly, by how much text is read and generated). Longer sessions with more tool use cost more.
  • Model tier matters. Opus is the most capable and most expensive. Sonnet is the default and a good balance. Haiku is the cheapest, best for simple tasks.
  • Check your spend. Type /cost during any session to see how much you've used so far.
  • Set limits. You can configure spending limits in your Anthropic account to avoid surprises.
  • Context = cost. Every file Claude Code reads, every tool it calls, and every response it generates adds to the token count. Keep sessions focused and use /compact to reduce context when a session gets long.
💡
For cost-conscious workflows, start with Sonnet (the default) and only switch to Opus for tasks that genuinely need deeper reasoning. Use /cost regularly until you develop an intuition for what sessions cost.

Get Started: Your First 30 Minutes

Five steps. By the end, you'll have Claude Code running, configured safely, and ready for your first real task.

1

Install Claude Code ~5 min

Claude Code is available in several forms. Pick the one that fits how you work:

Desktop App (easiest)

Download the Mac or Windows app. No terminal required. Best starting point for non-developers.

Web App

Run Claude Code in your browser at claude.ai/code. No install needed, works on any platform.

VS Code Extension

Search "Claude Code" in the VS Code Extensions marketplace. Integrates directly into your editor.

Terminal (npm)

For power users: npm install -g @anthropic-ai/claude-code, then run claude from any terminal.

ℹ️
Terminal install requirements: Node.js 18+. Install from nodejs.org if needed. The desktop app and web app have no prerequisites.
2

Open in the Right Folder ~3 min

⚠️
Most common beginner mistake: Opening Claude Code in a root or overly broad directory. It scans everything and gets confused. Always open at a specific project folder.

Do this:

  1. Open VS Code (or your preferred terminal)
  2. Navigate to a specific project folder: cd ~/my-projects/my-analysis
  3. Run claude and it launches with context from that folder only

Create one folder per project. Name it clearly. Claude Code uses the folder structure to understand scope and stay focused.

3

Configure Your Permissions ~5 min

Before your first session, set up a safe default config. This way Claude Code won't ask permission for every small thing, but it also won't have blanket access to your system.

Create or open: ~/.claude/settings.json

{
  "permissions": {
    "allow": [
      "Read",
      "Write(/Users/yourname/Projects/**)",
      "Bash(ls:*)",
      "Bash(echo:*)"
    ]
  }
}

Replace yourname with your Mac username. This allows reading any file, writing only inside your Projects folder, and running safe terminal commands. Everything else will ask for approval first.

💡
You can manage permissions conversationally: ask Claude Code "Please add the permission you just requested to my local config" and it will update the file for you.
4

Teach It Who You Are ~10 min

Create a CLAUDE.md file in your project folder (or globally at ~/.claude/CLAUDE.md). Claude Code reads it automatically every session, so you never re-explain your role, preferences, or data sources.

The more context you give it upfront, the better it performs from day one. See the full guide, templates, and seed prompts in the Teach It to Know You section below.

5

Run Your First Task ~10 min

Try something real. Here are three good first prompts depending on what you want to do:

If you want to analyze data:

I need to analyze last month's expense reports. I have CSV files in my Downloads folder. Can you start by listing what's there and suggesting how to combine them into a single summary?

If you want to automate a task:

I do this manually every week: [describe your process]. I want to turn this into a repeatable workflow. Let's start by walking through the steps so you understand the pattern.

If you just want to explore:

I'm new to Claude Code. What can I do from this folder? What files are here, and what would be a good first task to try together?

✅
You're set up. From here, explore the Workflows section for guided use cases, or browse the Skills section to find reusable automations.

Teach It to Know You

Three files that transform Claude Code from a generic assistant into one that knows your work.

💡
The core idea: Claude Code reads a handful of markdown files automatically at the start of every session. Fill them in once, and you never have to re-explain yourself again. These files are your "working memory" for the AI.

📋 Root CLAUDE.md

~/.claude/CLAUDE.md   or   ~/your-workspace/CLAUDE.md

The most important file in your setup. It tells Claude Code who you are, what you work on, where your data lives, and how you like to communicate. Without it, you re-explain yourself every session. With it, every session starts with full context.

# CLAUDE.md — [Your Name]

## Who I Am
Name: [Your name]
Title: [Your role]
Team: [Your team / department]
Focus: [1–2 sentences describing your work]

## Preferences
- Communication: [Direct / narrative / detailed]
- Output format: [Bullets / prose / tables]
- When in doubt, ask me before making changes to files

## Where Things Live
- Active projects: ~/projects/[name]/
- Permanent knowledge: ~/personal-context/
💡
Seed prompt: "Look at my CLAUDE.md and suggest what's missing. What context would make you more effective for someone in my role?"

👤 my-profile.md

personal-context/my-profile.md

Your role, background, and domain expertise. This helps Claude Code calibrate how to explain things — it won't over-explain basics you know cold, or assume expertise you don't have.

Include: your team's scope, the systems you work in, the data you rely on, common stakeholders, and anything that defines how you do your work.

💡
Seed prompt: "Ask me 5 questions and use my answers to write a my-profile.md for me."

✍️ my-writing-style.md

personal-context/my-writing-style.md

Your tone, format preferences, and phrases to avoid. Used when Claude Code drafts emails, summaries, or documents on your behalf. The output should sound like you, not like a generic AI.

Include: preferred register (formal/casual), bullet vs prose, things you never say, how you sign off emails, how detailed your summaries tend to be.

💡
Seed prompt: "Read the emails in [folder] and draft a writing-style.md that captures how I communicate."
ℹ️
You can also add a CLAUDE.md per project folder with details specific to that work: the goal, key data sources, open questions, and any domain gotchas. Update it as you learn. See the Workspace Structure section for how to organize these.

Guided Workflows

Step-by-step patterns for common non-developer use cases.

🔄 Workflow 1: Your First Session

▼

Goal: Get Claude Code open and run a real task end-to-end.

  1. Open a specific project folder in VS Code or terminal
  2. Run claude from within that folder
  3. Tell Claude Code what to work with: e.g., "Search my project files for..." or "Query the database for..."
  4. Review results before letting it make any changes to files
  5. Before ending your session, back up any important files

⚙️ Workflow 2: Recurring Task Automation

▼

Goal: Turn a repeated manual process into a one-command skill.

  1. Do the task manually once and note every step
  2. Write a CLAUDE.md describing the task, inputs, and expected outputs
  3. Run the task with Claude Code and let it learn the pattern
  4. Ask Claude Code to turn the workflow into a skill (skills/<name>/<name>.md)
  5. Test the skill on a new instance of the task
  6. Share with teammates if broadly useful

📊 Workflow 3: Data Analysis + Report

▼

Goal: Query data, analyze it, produce a formatted output (Sheets / Doc / Slides).

  1. Open Claude Code in your project folder
  2. Tell it where the data is: "Query [table/file] for [date range/criteria]"
  3. Ask for analysis, enrichments, or commentary on the result
  4. Ask Claude Code to format the output as a Google Sheet / Doc / Slides via Apps Script
  5. Review and share the output
💡
Tell Claude Code the destination upfront. Saying "I'm building this for a Google Doc" causes it to automatically embed the right formatting and structure.

🔔 Workflow 4: Building a Monitoring Alert

▼

Goal: Automatically check a condition and notify stakeholders.

  1. Define what you want to monitor (e.g., invoices not matched, approvals overdue)
  2. Ask Claude Code to write a script that queries the data and checks the condition
  3. Test the script manually and verify results
  4. Ask Claude Code to set up a cron job to run it on a schedule
  5. Add email or Slack notification as output

💬 Workflow 5: Formatted Slack Message

▼

Goal: Send a professional, well-formatted Slack announcement.

  1. Draft the content of your message (bullet points, key info)
  2. Tell Claude Code: "Using Slack Block Kit, format this as a Slack message: [paste content]"
  3. Claude Code returns Block Kit JSON
  4. Preview in Slack's Block Kit Builder (optional)
  5. Post to Slack

🌐 Workflow 6: Build a Website or Dashboard

▼

Goal: Publish an HTML page for documentation, reporting, or a prototype.

  1. Describe what the page should contain and who the audience is
  2. Ask Claude Code to build it as a self-contained HTML file
  3. Review the output in a browser
  4. Publish via GitHub Pages or paste into Google Sites
  5. Share the URL
💡
Non-technical users have built polished browser tools through purely conversational prompting. Just keep describing what you want and iterating.

📋 Workflow 7: Meeting Transcript to Action List

▼

Goal: Turn a meeting recording into an organized task list.

  1. Download or copy the transcript from your meeting
  2. Open Claude Code in your project folder
  3. Tell Claude Code: "Here is a meeting transcript -make me a checklist of action items"
  4. Review and refine the checklist
  5. Tell Claude Code: "Send these tasks to Asana" (requires Asana MCP)
  6. All action items land in your task manager with no manual entry

🧠 Workflow 8: Session with Memory (Notebook Brain)

▼

Goal: Use Claude Code across sessions without losing context.

  1. Create a personal-context/ folder with a CLAUDE.md
  2. Add domain knowledge, data definitions, and your profile to the folder
  3. At the start of each session, Claude Code reads the folder for context
  4. At the end of the session, run /wrapup to save what was learned
  5. Over time, the folder becomes richer and Claude Code gets more effective
personal-context/
  knowledge/         # Domain knowledge, business rules
  context/           # Table definitions, data dictionaries
  procedures/        # Runbooks, how-tos
  my-profile.md      # Who you are, your role, preferences
ℹ️
Built-in memory: Claude Code also has a native auto-memory system at ~/.claude/projects/*/memory/. It automatically saves things you ask it to remember across sessions. The manual personal-context/ approach above gives you more control and structure, but the built-in memory works with zero setup.

Best Practices

Distilled from community experience. The patterns that consistently produce the best results.

Folder & Context Management

  • Open at the right level. Always open in a specific project folder. Broad or root-level folders degrade performance and cause context noise.
  • Organize by project. Keep each project in its own folder with its own CLAUDE.md. Claude Code uses the folder structure to understand scope.
  • Keep CLAUDE.md files focused. More is not better. Keep instructions specific and concise, and cut anything that isn't actively useful.
  • Put the most important instructions at the TOP of your CLAUDE.md. Claude Code reads top-down, so context at the top carries more weight.
  • Keep files lightweight. Every session reads the folder contents. Bloated folders = slower, noisier sessions.

Prompting

  • Specify your knowledge source explicitly. Tell Claude Code whether to search the web, query a database, or read local files. Never let it guess.
  • Be precise about desired output format. If you want a Google Doc, a spreadsheet, or a Slack message, say so upfront.
  • For repeat tasks, build context files. Structured folders and MD files for recurring work (month-end processes, regular reports) make sessions dramatically more effective.
  • Tell Claude Code the destination. Say "I'm building this for Google Sites" and it will automatically embed the right style definitions and format output for that platform.
  • Use @ to attach files directly. Type @/path/to/file in your prompt to give Claude Code immediate access to a specific file. In the VS Code extension, you can also drag and drop files into the input.

Multi-Perspective Review

Before sharing any doc, proposal, or analysis, ask Claude Code to review it from multiple perspectives:

Review this from different perspectives:
(1) a senior expert who will question the technical details
(2) someone new to this area who doesn't know the jargon
(3) a skeptic who thinks we don't need this
(4) an executive who needs to understand the impact and risks

"Review this doc" → generic feedback. "Review this as someone who…" → specific, actionable issues you were too close to see.

Backing Up & Safety

⚠️
Back up regularly. Claude Code can unexpectedly delete or overwrite files. This is a known risk with any AI coding tool.
  • Checkpoint your work at every stable point. When something is "pretty good," commit or save it before continuing
  • Use git for version control. Pushing to a branch protects you from accidental overwrites
  • At the start of large tasks, explicitly ask Claude Code to save a backup first
  • Use hooks to add automatic guardrails. For example, you can configure a hook that auto-commits before Claude Code makes large edits, giving you a rollback point. Hooks are shell commands that run automatically before or after specific tool calls.

What NOT to Do

  • Don't open Claude Code at root or a broad directory level
  • Don't install plugins/MCPs liberally. Start with just what you need and add more as you go
  • Don't blindly install software Claude Code recommends. Understand what it does first
  • Don't give Claude Code blanket full permissions. Configure settings.json deliberately
  • Don't rely on Claude Code alone as your backup. Save important work externally too
  • Don't let Claude Code choose its knowledge source. Always specify where to look
  • Don't use tokens inefficiently. If a CLI is available for a tool, it's often more efficient than an MCP

Scheduling & Automation

Claude Code supports cron jobs for automated recurring tasks:

  • Daily news scans → automated market summary delivered each morning
  • Weekly status reports → auto-drafted from Slack/doc activity
  • Alert monitoring → automatic notification when conditions are met

Ask Claude Code: "Set up a cron job to run this script every Monday at 8am"

Reducing Token Usage

  • Type /context to see a graphical view of context window usage
  • Type /cost to see token cost so far in the session
  • Start new sessions when switching topics to keep context clean and focused
  • Use dynamic MCP loading (experimental). It loads tool descriptions on-demand instead of preloading everything, which can reduce context by ~40k tokens per session

Session Length and Context

  • Sessions have a ceiling. Claude Code has a finite context window. As a session gets long, older messages are automatically compressed to make room. You may notice it "forgetting" earlier details.
  • Use /compact proactively. This compresses your current session context, freeing up space and reducing cost. Good to run after completing a major chunk of work.
  • Start fresh when the task changes. If you're switching to a different project or topic, open a new session rather than continuing a long one. Clean context produces better results.
  • Use /resume to pick up where you left off. If you close a session accidentally or want to return to earlier work, /resume reloads the previous session state.
  • Watch for drift. In very long sessions, Claude Code may start repeating itself or losing track of earlier decisions. That's a sign to compact or start fresh.

Integrations (MCPs)

MCPs (Model Context Protocol) give Claude Code access to external tools and data sources, so you can interact with them in natural language.

Available MCPs

IntegrationWhat it doesNotes
GitHub Search repos, read files, create/review PRs, manage issues from natural language Available via MCP servers repo
Slack Read channels, search messages, post updates, pull activity summaries Check availability for your org
Google Drive Read, create, search, and share files in Drive; also enables Docs/Sheets integration Community MCP available
Gmail / Calendar Access email and calendar events from Claude Code Google Workspace CLI is an efficient alternative
Filesystem Read, write, and organize local files with configurable access controls Built-in; configure paths in settings.json
Web Search / Fetch Browse the web, fetch page content, search for current information Built-in capability
ℹ️
Browse the full MCP ecosystem at the MCP servers repository. Anthropic maintains official servers for many common tools, and the community adds more regularly.

MCPs vs. CLIs

  • MCPs are convenient but consume more tokens because they expose broad tool surfaces
  • CLIs can be more efficient for specific data access since they fetch exactly what's needed
  • For high-volume or token-sensitive workflows, consider whether a CLI approach is better
  • Start with the core MCPs you need. Resist installing many at once

Troubleshooting MCPs

MCPs failing or not connecting

▼
  1. Change model: type /model and select a different version
  2. Type /mcp and manually restart each MCP one by one

MCP still broken after restart

▼
  1. Check your MCP config in ~/.claude/settings.json for typos or bad paths
  2. Make sure any required environment variables (API keys, tokens) are set
  3. Try removing and re-adding the MCP configuration
  4. Check the MCP server's own logs for errors

FAQ

Common questions from the community.

What is Claude Code?

▼
Claude Code is Anthropic's AI coding and analysis assistant that runs in VS Code or any terminal. It can access external tools (databases, Jira, Drive, etc.) via MCPs and can automate complex, multi-step tasks through natural language instructions.

How do I install it?

▼ 
npm install -g @anthropic-ai/claude-code
claude
Requires Node.js 18+. You can also install the VS Code extension from the Extensions marketplace for an integrated IDE experience.

Can I use Claude Code without VS Code?

▼
Yes. Once installed, run claude directly from any terminal window. VS Code is a convenient wrapper with additional features, but not a requirement.

Why is Claude Code slow or confused?

▼
Most likely you opened it at too broad a folder level. Always launch Claude Code from a specific project folder, not your root or home directory. The broader the folder, the more context noise it picks up.

Claude Code stopped responding. What do I do?

▼
This is sometimes an authentication issue. Restart the application. Use /resume to recover your previous session.

How do I switch between models?

▼
Run /model to see available models and switch. Default is Sonnet. Tradeoffs:
  • Opus: Smartest reasoning, best for complex or multi-step tasks
  • Sonnet: Good default balance of quality and speed
  • Haiku: Faster and cheaper, good for simple or high-volume tasks
Use /fast to toggle faster output on the same model tier (it does not switch to a cheaper model). To set a permanent default, add the model preference to your global CLAUDE.md file.

Why is Claude Code giving wrong or hallucinated answers?

▼
Likely because you didn't specify which knowledge source to use. Always tell Claude Code explicitly: use web search, query the database, use this specific file, or look in Glean. Failure to specify is the #1 cause of hallucinated results.

How do I build a reusable skill?

▼
Iterate on a task in a normal conversation until you're happy with the output. Then say: "I want to do this in a repeatable way -can you turn it into a skill?" Claude Code will create a skill file in the appropriate directory. See the Skills section above for the full file structure.

How do I set up automated scheduling (cron jobs)?

▼
Ask Claude Code to set up a cron job for you: "Set up a cron job to run this script every weekday at 8am." It will write the cron entry and configure it. Use custom commands for recurring task triggers.

Claude Code seems to have deleted my work. What happened?

▼
Claude Code can unexpectedly delete or overwrite files, which is a known risk with AI coding tools. Always back up your work regularly. Use git to protect against accidental overwrites, and commit at every stable checkpoint before continuing complex tasks.

Should I install a lot of MCPs?

▼
Be conservative. Start with the core integrations you actually need. Extra plugins have caused the most technical issues for non-developer users (launch failures, authentication errors, etc.). Add integrations one at a time and validate each one works before adding more.

What is the /insights command?

▼
Provides personalized feedback on how you interact with Claude Code, helping you identify prompting patterns to improve and optimize your workflows over time.

What can't Claude Code do?

▼
  • Browse or click through GUIs. It can't see your screen or interact with desktop apps unless you set up a computer-use MCP.
  • Remember across sessions automatically. Without a CLAUDE.md or the built-in memory system, every session starts from zero.
  • Access internal company systems. It can't reach your company's Jira, Slack, databases, or internal tools unless you wire up an MCP or CLI for each one.
  • Run indefinitely. Sessions have context limits. Very long sessions degrade in quality as older context gets compressed.
  • Guarantee correctness. It can produce plausible but wrong results, especially with data analysis. Always validate outputs against your domain knowledge.

How do I share something I built?

▼
  • Websites / HTML: Host on GitHub Pages via the /publish-html-github skill, or paste into Google Sites
  • Skills / scripts: Share via a private GitHub repo or team folder
  • Broader sharing: Publish as a community plugin
