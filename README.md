# Claude Code PKM Guide

A practical field guide to using Claude Code as a personal knowledge management system. Built for non-developers in finance, operations, audit, and anyone who wants to automate, analyze, and build without writing code.

## What's in the guide

A single self-contained HTML file (`claude-pkm-guide.html`) covering:

- **What is Claude Code** and how it differs from other AI tools
- **Cost and billing** basics (pay-per-token, model tiers, spending controls)
- **Getting started in 30 minutes** with multiple install options (desktop app, web app, VS Code, terminal)
- **Teaching Claude Code your context** with CLAUDE.md files and personal profiles
- **Workspace structure** for organizing projects and persistent knowledge
- **The knowledge cycle** across daily sessions, project lifecycles, and long-term learning
- **8 guided workflows** from first session through recurring automation, data analysis, monitoring alerts, and session memory
- **Best practices** for prompting, safety, token management, session length, and scheduling
- **MCP integrations** for connecting external tools (GitHub, Slack, Google Drive, Gmail, databases)
- **Skills library** for building and sharing reusable slash commands
- **FAQ** including limitations, model selection, troubleshooting, and sharing your work

## How to use

Open `claude-pkm-guide.html` in any browser. No build step, no dependencies, no server required.

To host it: push to GitHub and enable GitHub Pages, or paste into any static hosting service.

## Roadmap

Planned improvements for future versions:

- **Obsidian integration guide.** How to use Obsidian as the knowledge layer alongside Claude Code and VS Code. Obsidian handles note-taking, linked thinking, and long-term knowledge storage. Claude Code handles automation, analysis, and tool execution. VS Code is the bridge. The guide will cover folder structure, plugin recommendations, and how the three tools complement each other.
- **PARA + BASB framework.** Implementing Tiago Forte's Building a Second Brain methodology and PARA (Projects, Areas, Resources, Archive) organizational framework within the Claude Code PKM workflow. This includes mapping PARA categories to workspace folders, using the knowledge cycle to feed the BASB capture/organize/distill/express pipeline, and practical templates for each stage.
- **Before-and-after examples.** Real-world walkthroughs showing a manual process next to the Claude Code version, with actual prompts and outputs.
- **Advanced patterns.** Subagents for parallel work, hooks for automated guardrails, and multi-project context management.
